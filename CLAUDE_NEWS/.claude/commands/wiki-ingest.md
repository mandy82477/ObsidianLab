# Wiki Ingest

讀取今日日報，更新 wiki 知識庫。每天在新聞聚合器跑完後執行。

## 步驟

### 1. 確認今日日報

讀取 `news/$ARGUMENTS.md`（若無提供日期，使用今天的日期 YYYY-MM-DD）。
若檔案不存在，停止並告知使用者。

### 2. 載入 wiki 現況

同時讀取：
- `wiki/index.md` — 取得所有現有頁面清單
- `wiki/log.md` — 確認最近是否已處理過同一份日報（避免重複 ingest）

### 3. 判斷受影響的頁面

比對日報內容與 index.md 中的實體和議題，找出：
- 日報中提到的既有 entities（例如模型名稱、產品、人物）
- 日報中提到的既有 topics（例如定價爭議、功能趨勢）
- 日報中出現但尚無專頁的新實體或新議題

### 4. 更新既有頁面

對每個受影響的頁面：
- 在「歷史記錄」或「時序」區塊最上方插入新事件（最新在上）
- 若現況有變化，更新「現況」或「摘要」段落
- 若為 topics/ 頁面，在「技術彙整」區塊累積新的技術細節：
  - 涉及的技術機制、API 行為、參數變化
  - 已確認的限制或已知問題
  - 社群發現的 workaround 或最佳實踐
  - 只新增尚未記錄的條目，不重複
- 同步更新「最後更新」欄位為今日日期

格式規範見 `CLAUDE.md`。

### 5. 建立新頁面（視需要）

**新 topics/ 頁**：若某個議題在日報中出現，且 log.md 顯示昨天也出現過同一議題，則建立新的 topics/ 頁。

**新 entities/ 頁**：若某個實體被日報明確描述（有具體資訊可記錄），建立新的 entities/ 頁。

頁面格式嚴格遵守 CLAUDE.md 中的模板。

### 6. 更新 log.md

在 `wiki/log.md` 末尾 append 一筆記錄（不可修改既有條目）：

```
## YYYY-MM-DD Ingest

- 來源日報：[[news/YYYY-MM-DD]]
- 更新頁面：（列出修改的檔案）
- 新增頁面：（列出新建的檔案，若無則寫「無」）
- 摘要：（一句話說明今日主要新聞方向）
```

### 7. 更新 wiki/index.md

若有新增頁面，在 index.md 對應分類下補上連結。
格式與現有條目保持一致。

## 注意事項

- 繁體中文為主，英文術語保留英文
- 每次修改頁面都必須更新「最後更新」欄位
- 若日報今日無新內容（來源全部失敗），在 log.md 記錄一筆「無新內容」即可，不需更新其他頁面
- `news/` 目錄為唯讀，不可修改日報內容
